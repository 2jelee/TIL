# 2️⃣ 핵심 개념 정리 (What)

## 🔹 개념 1

## 연결 리스트 (Linked List)

### 정의

연결 리스트는 **노드(Node)** 단위로 데이터를 저장하는 선형
자료구조이다.\
각 노드는 **데이터 + 다음 노드의 참조값(주소)** 를 가진다.

배열과 달리 **메모리 상에서 연속적으로 배치되지 않는다.**

------------------------------------------------------------------------

## 🔹 단일 연결 리스트 (Singly Linked List)

### 정의

각 노드가 **자기 다음 노드만 가리키는 구조**

Head → \[data \| next\] → \[data \| next\] → \[data \| null\]

### 핵심 포인트

-   Head 접근: O(1)
-   중간 탐색: O(n)
-   삽입/삭제 시 "뒤를 밀 필요 없음"
-   메모리 사용량: next 포인터 1개

### 왜 중요한가?

-   삽입/삭제가 빈번한 경우 배열보다 효율적
-   큐/스택 구현의 기반 구조

------------------------------------------------------------------------

## 🔹 이중 연결 리스트 (Doubly Linked List)

### 정의

각 노드가 **이전(prev) + 다음(next)** 을 모두 참조하는 구조

null ← \[prev \| data \| next\] ⇄ \[prev \| data \| next\] ⇄ \[prev \|
data \| next\] → null

### 핵심 포인트

-   양방향 탐색 가능
-   삭제 시 이전 노드를 찾기 위한 재탐색 불필요
-   메모리 사용량 증가 (포인터 2개)

### 왜 중요한가?

-   브라우저 뒤로가기
-   LRU Cache 구현
-   양방향 순회가 필요한 구조

------------------------------------------------------------------------

## 🔹 원형 연결 리스트 (Circular Linked List)

### 정의

마지막 노드가 다시 첫 노드를 가리키는 구조

Head → A → B → C\
↑ ↓\
← ← ← ←

### 핵심 포인트

-   끝이 존재하지 않음
-   순환 구조
-   라운드 로빈 스케줄링에 활용

------------------------------------------------------------------------

# 3️⃣ 내부 동작 원리 (How)

## 구조 설명

연결 리스트는 **포인터(참조)** 기반 구조이다.

-   배열: index 기반 접근
-   연결 리스트: 참조를 따라 이동

------------------------------------------------------------------------

## 동작 과정 단계별 정리

### 1️⃣ 삽입 (중간 위치)

1.  해당 위치 이전 노드 탐색
2.  newNode.next = current.next
3.  current.next = newNode

------------------------------------------------------------------------

### 2️⃣ 삭제

1.  삭제 대상 이전 노드 탐색
2.  previous.next = target.next

------------------------------------------------------------------------

### 3️⃣ 탐색

1.  head부터 시작
2.  next를 따라 이동
3.  null 또는 목표 값까지 반복

------------------------------------------------------------------------

# 4️⃣ 시간복잡도 / 공간복잡도

연산         단일 연결리스트   이중 연결리스트   배열
  ------------ ----------------- ----------------- ----------
탐색         O(n)              O(n)              O(1)
맨 앞 삽입   O(1)              O(1)              O(n)
맨 뒤 삽입   O(n)              O(1)\*            O(1)\*\*
중간 삽입    O(n)              O(n)              O(n)
삭제         O(n)              O(n)              O(n)
공간복잡도   O(n)              O(n)              O(n)
