# [TIL] Spring Boot 웹 개발 기초 핵심 정리

### 1. Gradle (빌드 자동화 도구)
* **개념**: Java 소스 코드를 설정에 맞게 자동 빌드하여 실행 가능한 파일(`.jar`)로 변환.
* **build.gradle**: 빌드 스크립트 파일(Groovy/Kotlin 사용). 라이브러리 의존성 및 프로젝트 설정 관리.

### 2. 네트워크 및 HTTP 기초
* **Client/Server**: 브라우저(클라이언트)의 요청(Request)에 서버가 응답(Response)하는 구조.
* **IP & Port**: 서버 위치 주소(IP)와 특정 프로세스를 구분하는 통로(Port).
* **HTTP**: 데이터를 주고받는 통신 규약.
    * **Method**: GET(리소스 조회), POST(데이터 생성/게시) 등.
    * **구성**: Header(메타 데이터), Body(실제 데이터).

### 3. Spring MVC 패턴
* **Model**: 데이터 및 비즈니스 로직 담당. DB 연동.
* **View**: 사용자 인터페이스(UI) 담당.
* **Controller**: 모델과 뷰 사이의 상호작용 제어. 사용자 요청 처리.
* **DispatcherServlet**: 모든 요청을 분석하여 적절한 컨트롤러에 전달하는 Front Controller 역할.

### 4. Controller 활용
* **@Controller**: 해당 클래스가 컨트롤러임을 명시.
* **@RequestMapping**: 중복되는 URL 경로를 클래스 단위로 공통 관리.
* **@GetMapping, @PostMapping**: HTTP Method별 API 경로 매핑.
* **@ResponseBody**: 메서드 반환 값을 뷰 탐색 없이 응답 본문에 직접 전달.

### 5. 정적/동적 페이지
* **정적 페이지**: `resources/static` 위치. 서버 가공 없이 HTML 파일 그대로 반환.
* **동적 페이지**: `resources/templates` 위치. 템플릿 엔진(Thymeleaf 등)이 Model 데이터를 HTML에 적용하여 생성 후 반환.

### 6. 개발 도구 및 설정
* **Lombok**: `@Getter`, `@Setter` 등으로 반복적인 자바 코드(Boilerplate) 자동 생성.
* **application.properties**: 서버 포트, DB 연결 등 스프링 설정값 관리.
* **MySQL**: 관계형 데이터베이스(RDBMS).
* **Postman**: 작성한 API의 동작 여부를 테스트하는 도구.

### 7. 테스트 코드
* **필요성**: 소프트웨어 버그 방지 및 예상한 대로 동작하는지 검증.
* **JUnit**: 자바 단위 테스트 프레임워크. `@Test`, `@DisplayName`, `Assertions` 사용.
