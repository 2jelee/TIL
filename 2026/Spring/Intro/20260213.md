### 중요한 포인트
- 의존성 주입 (3가지)  // TODO: 코드 예시 확인하기
  - 1. 필드에 직접 주입
  - 2. 메서드를 통한 주입
  - 3. 생성자를 통한 주입
- 제어의 역전 : ex) 기존 Consumer -> Food로 흘렀음. 추가적인 코드 변경 없이 제어의 흐름이 Food -> Consumer로 역전.
- 강한 결합 해결방법
    - 1. 각 객체에 대한 객체 생성은 딱 1번만!
    - 2. 생성된 객체를 **모든 곳**에서 재사용!
    - 3. **생성자 주입**을 사용하여 필요로 하는 객체에 해당 객체 주입!

### 인상 깊은 문장 / 내용
> 제어의 역전 (IoC: Inversion of Control)  
> 기존 제어의 흐름은 Controller ⇒ Service ⇒ Repository 였으나,  
> DI 즉, 의존성 주입을 통해 제어의 흐름을 Repository ⇒ Service ⇒ Controller 로 역전함으로써 효율적인 코드로 바뀜.
